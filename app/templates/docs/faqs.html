Frequently Asked Questions


FAQ: I've run a test and it's not completed or run into an error. What should I do?

Most likely, the error is due to incorrect input data. Please check your input data thoroughly to ensure it is in the correct format. A common error we've observed involves the "Spill Volume (mÂ³)" column, where the values are mistakenly entered as datetime values instead of volume values. Please double-check your input data for such discrepancies and correct them to prevent similar issues in future tests.


FAQ: If you close the application while a test is running, what happens?

If the application is closed midway through a run, say because your computer crashes or you accidentally close the application, the test will not be saved. You also may have to run ./setup.bat in order reconnect the database and then ./run.bat in order to get the application running properly again


FAQ: My visualisation is running slowly, what can I do?

As the visualisation has to handle a lot of data points, it can sometimes run slowly. If you are experiencing this issue, try changing the interval button from 15m to 1hr. This means the visualisation has less data to load and so should run faster.


FAQ: I'm trying to delete a run or a test but its not deleting, what should I do?

If theres an issue with deleting a run or a test, try running ./setup.bat again in order to reconnect the database. Most issues can be solved by running this command and then re running the application with ./run.bat


FAQ: I've just reset the database by running setup again and i still see old runs and locations in my sidebar, what do i do?

If you've reset the database and still see old runs and locations in your sidebar, just simply reset your cookies (Click the favicon left of the website URL and you should a clear cookies button). This should refresh the sidebar and show the correct data.
